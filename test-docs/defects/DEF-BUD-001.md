\# DEF-BUD-001 预算提醒阈值为 0 时被错误接受



\## 基本信息



\- 缺陷编号：DEF-BUD-001

\- 关联测试用例：TC-BUD-007

\- 所属模块：预算统计模块

\- 缺陷类型：输入校验 / 边界值处理

\- 严重程度：Medium

\- 优先级：High

\- 当前状态：Open



\## 缺陷描述



当用户将预算提醒阈值设置为 0 时，系统未按照需求拒绝该输入，而是将其作为有效数据处理。



提醒阈值的有效范围应为 1% 至 100%，0 属于无效边界值。



\## 前置条件



1\. 进入预算设置功能。

2\. 设置预算类型为月度预算。

3\. 设置周期为 2026-07。

4\. 设置总预算金额为 6000。



\## 测试步骤



1\. 将提醒阈值设置为 0。

2\. 提交预算数据。

3\. 检查预算校验结果。



\## 测试数据



\- budgetType: month

\- period: 2026-07

\- amount: 6000

\- threshold: 0



\## 预期结果



系统拒绝保存。



校验结果：



```text

valid = false

```



并提示：



```text

提醒阈值需在 1% 到 100% 之间

```



\## 实际结果



系统接受该输入。



自动化测试结果：



```text

Expected: false

Received: true

```



\## 测试方法



边界值分析法。



阈值有效范围为 1% 至 100%，选取下边界外值 0 进行测试。

\## 初步原因分析



预算阈值标准化过程中，数值 0 被作为 falsy 值处理，并被替换为默认阈值。



因此后续校验未获取到用户实际输入的 0。
