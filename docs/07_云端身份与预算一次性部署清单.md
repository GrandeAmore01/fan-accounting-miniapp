## 二、TiDB Cloud 一次性执行

在连接当前生产数据库的 SQL Editor 中，完整执行：

`server/sql/cloud_user_budget_migration.sql`

该脚本会：

1. 为 `users` 表增加唯一 `openid` 字段。
2. 新建 `budgets` 表，其中 `user_id` 显式使用 `utf8mb4_bin`，与线上 `users.user_id` 保持一致。
3. 建立预算与用户的外键及用户周期唯一约束。

执行完成后运行：

```sql
USE fan_accounting;

SHOW COLUMNS FROM users LIKE 'openid';
SHOW TABLES LIKE 'budgets';
SHOW INDEX FROM budgets;
```

预期结果：`users.openid` 存在，`budgets` 表存在，并包含 `PRIMARY` 和 `uk_budgets_user_period` 索引。

数据库验证：

```sql

SELECT user_id, openid, login_status, created_at
FROM users
WHERE openid IS NOT NULL
ORDER BY created_at DESC;

FROM budgets
ORDER BY updated_at DESC;
```

## 五、注意事项

- `WECHAT_APP_SECRET` 和 `AUTH_TOKEN_SECRET` 不得写入代码、截图或 Git。
- 原 `local-user` 云端业务数据不会自动归属给真实微信用户。
- 当前设备的本地预算会在真实用户云端预算为空时自动上传一次。
- 本地缓存按真实微信用户分别存储，不同微信用户不会共用同一份缓存。
- 消费、预算、藏品和舞台以云端为准；本地仅保存最近一次成功数据和临时草稿。
- 如果 SQL 提示 `Duplicate column name 'openid'`，先运行验证查询；确认字段和唯一索引已经存在后，只执行脚本中的 `CREATE TABLE budgets` 部分。
