# 06_变更申请

本文件夹用于记录开发过程中对 01、02、03 初稿中功能、逻辑、字段、页面或测试要求的变更。

当实际开发内容与原本文档不一致，或新增、删除、调整了功能时，请同学们复制 `变更申请模板.md` 新建一份变更记录，并按顺序编号。

## 命名建议

- `C001_预算统计仅支持月度预算.md`
- `C002_藏品图鉴新增筛选字段.md`
- `C003_消费记录保存逻辑调整.md`

## 填写要求

1. 一个变更写一个文件，不要把多个无关变更写在同一份记录里。
2. 变更编号按 `C001`、`C002`、`C003` 递增。
3. 如果不确定原逻辑、影响范围或测试方式，可以写“不清楚”或“不确定”，但不要空着。
4. 涉及页面、文件、数据字段、用户操作流程的变更，要尽量写具体。
5. 状态更新时，请同步修改【当前状态】。

## 状态说明

- 提出中：发现变更，还没有确认是否接受。
- 已同意：组内确认可以按这个变更开发或保留。
- 已开发：代码已经完成，但还没有完成测试确认。
- 已测试：已经按测试方式验证通过。
- 已放弃：该变更不再采用。
